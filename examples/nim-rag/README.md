# Evaluating a NIM-powered RAG agent with OpenGATE

A worked example: build a small retrieval-augmented (RAG) agent whose answers are generated by an **NVIDIA NIM** model, then verify those answers with **OpenGATE** — deterministically, with **no LLM judge**.

NIM ([NVIDIA Inference Microservices](https://build.nvidia.com)) serves models behind an OpenAI-compatible API, so the agent is a thin wrapper. The interesting part is the evaluation: OpenGATE checks each answer against gold with pure logic — the required facts are present, every number traces back to the retrieved context, and when the context can't answer, the agent abstains rather than fabricates. No second model, no API cost, same verdict every run.

> **The point of the example:** OpenGATE isn't tied to any one model or framework. Point it at a NIM-powered agent, a hosted API, or a local pipeline and the grounding check is identical — which is exactly the kind of framework-agnostic, pipeline-fits-anywhere evaluation NVIDIA encourages developers to build.

## Prerequisites

A free **NVIDIA API key** from [build.nvidia.com](https://build.nvidia.com) (it exposes NIM models through an OpenAI-compatible endpoint):

```bash
export NVIDIA_API_KEY=nvapi-...
# optional:
export NIM_MODEL=meta/llama-3.1-8b-instruct              # any NIM chat model
export NIM_BASE_URL=https://integrate.api.nvidia.com/v1  # or a self-hosted NIM container
```

## Two ways to run it

### 1. Python notebook — the interactive showcase

[`python/evaluate_nim_rag.ipynb`](python/evaluate_nim_rag.ipynb) builds the full RAG loop (a minimal lexical retriever + NIM generation) and scores every answer with the [`opengate-grounding`](https://pypi.org/project/opengate-grounding/) package:

```bash
cd python
pip install -r requirements.txt
jupyter lab evaluate_nim_rag.ipynb   # or: jupyter notebook
```

It uses `check_grounding()` to grade each answer, `assert_grounded()` to show a CI-style gate, and notes the DeepEval `GroundingMetric` drop-in. Without an API key it falls back to canned answers so you can see the evaluation mechanics end-to-end before wiring NIM.

### 2. Node — the full framework + CI regression gate

[`node/`](node) wraps a NIM adapter ([`nim-adapter.mjs`](node/nim-adapter.mjs)) in the OpenGATE runner. The gold cases live in [`node/datasets/cases/`](node/datasets/cases); each supplies the retrieved `context` (your retriever's output), the facts a correct answer must contain, and — for one case — an unanswerable question the agent must decline.

```bash
cd node
NVIDIA_API_KEY=nvapi-... \
npx --yes @pharmatools/opengate \
  --adapter ./nim-adapter.mjs --datasets ./datasets --online --ci
```

`--ci` exits non-zero if any answer drops a required fact, invents a number, or fails to abstain — so a model or prompt change that starts hallucinating fails the build. The `grounding` scorer is the one that runs here; the other bundled scorers no-op because this adapter only implements the grounding capability. Commit a baseline (`--baseline`) to also catch metric regressions over time.

## What each case checks

| Case | Question | Tests |
|------|----------|-------|
| `refund` | refund window + restocking fee | required facts present, no invented numbers |
| `shipping` | free-shipping threshold + delivery time | facts present; `$50`/`$4.99` allowed, other numbers flagged |
| `warranty` | warranty length + scope | required facts present |
| `enterprise-price` (unanswerable) | annual Enterprise price (absent from context) | agent **abstains** instead of fabricating |

## How the boundary is drawn

The `grounding` scorer hands the adapter a question and the **already-retrieved context**, and checks the generated answer against it. That isolates the generation step — where hallucinations actually appear — from retrieval. In the notebook you can see the retriever run too; in the Node gold set, the `context` field stands in for your retriever's output so the check is reproducible. To evaluate retrieval quality itself, use OpenGATE's separate `retrieval` capability.
